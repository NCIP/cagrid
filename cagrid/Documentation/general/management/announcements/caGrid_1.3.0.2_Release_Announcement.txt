On behalf of the CBIIT Quality Assurance Team, caGrid Knowledge Center, and caGrid Development Team, I am pleased to announce the release of caGrid version 1.3.0.2. This is a regression-tested maintenance release of the caGrid 1.3 stream, and is recommended for all caGrid 1.3 users.  

Bugs Fixed Since 1.3.0.1 Release:
  * FQP erroneously indicates "COMPLETE_WITH_ERROR" state when a target data service returns no results.
  * CQL queries making use of EQUAL_TO as the default predicate value will fail
  * DQL queries making use of schema-specified default values may cause errors
  * SDK41QueryProcessor does not set the password for non-static logins
  * SDK 4.1 backed data services using Local API use the "wrong" bean proxy implementation
  * Domain models generated from XMI for caCORE SDK 4.x don't preserve caDSR IDs on Attributes
  * Certain types of Hibernate database tables for subclasses are not supported by Data Services with SDK 4.0 and 4.1
  * MMS UMLAssociationExclude wildcard character as default value not honored 
  * Infinite loop in Introduce updater

Please see the release notes for a list of detailed changes: 
  https://gforge.nci.nih.gov/frs/shownotes.php?release_id=3671
